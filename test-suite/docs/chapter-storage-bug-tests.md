# 章节存储功能测试用例

## 概述

本文档针对章节存储功能设计测试用例，重点验证以下用户反馈的问题修复：
1. 第一章的文案开始没有加载出来
2. 显示有文字，但是是空的
3. 第一章第二章文案颠倒的问题
4. 第二章写的文案缺存到了第一章

### 问题根因

通过 `useChapterManager.ts` 分析，问题出在初始化阶段的竞争条件：
- `ChapterList` 和 `useChapterManager` 各自独立调用 `refreshChapters()`
- 两者都监听 `creatorai:chapterSelected` 事件
- 初始化时的 `load` 事件与用户切换事件混在一起，导致状态不一致

### 修复方案

通过检查 `cause` 字段区分事件来源：
- `cause === "load"`：初始化时的自动加载，useChapterManager 不处理
- `cause === "user"`：用户点击切换章节
- `cause === "create"`：用户创建新章节
- `cause === "delete"`：用户删除章节
