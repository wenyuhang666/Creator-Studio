## 1. AI 对话功能测试

### TC_CHAT_001: 基本对话功能

| 属性 | 值 |
|------|-----|
| 用例编号 | TC_CHAT_001 |
| 用例标题 | 基本对话功能 |
| 测试类型 | 功能测试 |
| 优先级 | **P0** (阻塞) |
| 关联需求 | REQ_AI_CHAT_001 |

**测试前提**
- AI 引擎已构建 (`npm run ai-engine:build`)
- 测试环境配置了有效的 API Key (TEST_API_KEY)
- 测试 API 端点可用

**测试步骤**
1. 发送简单的问候消息 "你好"
2. 调用 AI 引擎的 chat 接口
3. 等待并解析响应

**预期结果**
- 返回 `type: "done"` 的响应
- `content` 字段包含 AI 的回复
- 回复内容长度 > 0

**测试数据**
- 输入: `{ messages: [{ role: "user", content: "你好" }] }`
- 期望输出: `{ type: "done", content: "...", toolCalls: [] }`

cd packages/ai-engine
bun install

cd test-suite

node run.mjs ai-engine-spawn

### B. 预期错误码

| 错误码 | 含义 | 可能原因 |
|-------|------|---------|
| 401 | 未授权 | API Key 无效或为空 |
| 403 | 禁止访问 | 权限不足 |
| 404 | 未找到 | API 端点错误 |
| 429 | 请求过多 | 超出速率限制 |
| 500 | 服务器错误 | AI 服务端问题 |
| timeout | 超时 | 网络问题或服务无响应 |

### C. 测试用例优先级说明

| 优先级 | 说明 | 测试策略 |
|-------|------|---------|
| P0 (阻塞) | 核心功能不可用 | 必须通过，否则阻塞发布 |
| P1 (重要) | 重要功能有问题 | 应该通过，建议修复 |
| P2 (优化) | 体验优化类问题 | 可接受失败，持续改进 |
