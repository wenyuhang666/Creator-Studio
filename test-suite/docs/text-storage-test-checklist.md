# 快速测试指南 - 文本编辑和文件存储功能

## 测试前提

1. 启动 Creator Studio 应用程序
2. 创建一个测试项目或打开已有项目
3. 创建至少一个章节

---

## 第一轮：核心功能测试（15 分钟）

### 1. 保存指示灯状态测试 (5分钟)

#### 测试 1：未保存状态
```
步骤：
1. 确认章节处于已保存状态（状态栏显示 ✓ 已保存 绿色）
2. 在编辑器中修改一些文字
3. 观察状态栏变化

预期：状态栏变为 ● 未保存 (红色/橙色圆点)
```

#### 测试 2：保存中状态
```
步骤：
1. 修改章节内容后立即按 Ctrl+S
2. 快速观察状态栏变化

预期：状态栏显示 ⟳ 保存中... (旋转图标)
```

#### 测试 3：已保存状态
```
步骤：
1. 等待保存完成或再次按 Ctrl+S
2. 观察最终状态

预期：状态栏显示 ✓ 已保存 (绿色勾选)
```

### 2. 手动保存功能测试 (5分钟)

```
步骤：
1. 修改章节内容
2. 观察状态栏从 ● → ⟳ → ✓ 的变化过程
3. 用文件管理器打开项目目录
4. 检查 chapters/ 目录下的 .txt 文件
5. 确认文件内容与编辑器一致
6. 检查 chapters/index.json 的 updated 字段已更新
```

### 3. 自动保存功能测试 (5分钟)

```
步骤：
1. 确保章节处于已保存状态（状态栏 ✓）
2. 修改章节内容
3. 不要做任何操作，观察状态栏
4. 等待约 2 秒

预期：自动触发保存，状态栏 ● → ⟳ → ✓

步骤：
5. 不做任何修改
6. 等待 5 秒

预期：状态栏保持 ✓，不触发保存
```

---

## 第二轮：导出和文件夹功能测试 (10分钟)

### 4. TXT 导出功能测试

#### 单个章节导出
```
步骤：
1. 选择要导出的章节
2. 点击导出按钮（菜单或工具栏）
3. 选择"导出当前章节"
4. 选择保存位置
5. 确认导出

验证：
- 检查导出的文件是否存在
- 检查文件内容是否正确
- 检查编码是否为 UTF-8
```

#### 批量导出
```
步骤：
1. 选择"导出所有章节"
2. 选择导出文件夹
3. 确认导出

验证：
- 检查文件夹中的文件数量
- 检查每个文件内容是否正确
```

### 5. 打开文件夹功能

```
步骤：
1. 确认状态栏右侧有"打开文件夹"按钮
2. 选择一个章节
3. 点击"打开文件夹"按钮

预期：
- 文件管理器打开
- 自动定位到章节文件所在位置
- 文件被选中高亮显示
```

---

## 第三轮：边界和异常测试 (10分钟)

### 6. 边界条件测试

```
空内容测试：
1. 清空章节所有内容
2. 保存
3. 重新加载
预期：内容保持为空，字数为 0

特殊字符测试：
1. 输入：测试🌟✨💡 《》【】 ①②③
2. 保存并重新加载
预期：内容完全一致，无乱码
```

### 7. 连续操作测试

```
快速保存测试：
1. 连续快速按 Ctrl+S 3-4 次
2. 观察状态栏
预期：系统正常处理，无异常

切换章节测试：
1. 修改章节1并保存
2. 切换到章节2
3. 切换回章节1
预期：章节1内容保持不变
```
