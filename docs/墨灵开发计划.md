# 桌宠化AI写作助手（墨灵）开发计划

## 一、架构分析

### 1.1 创意方案与现有架构的契合点

| 创意方案功能 | 现有架构支撑 | 实现方式 |
|-------------|-------------|---------|
| 桌宠停靠组件 | MainLayout + CSS Grid | 新增 PetLayer 覆盖层 |
| 状态系统 | Zustand store | 新建 petStore |
| AI 会话能力 | lib/ai.ts + AIPanel | 复用现有 AI 能力 |
| 章节状态 | useChapterManager | 监听章节变化触发桌宠行为 |
| 写作统计 | StatusBar + wordCount | 复用或扩展 |
| 灵感气泡 | 新建组件 | Popup/Bubble 机制 |

### 1.2 技术决策

**技术选型理由：**
- **React 19** + **TypeScript**：与现有项目一致
- **Zustand**：现有项目已使用，便于状态共享
- **Ant Design 6**：现有项目已使用，继续使用其组件
- **CSS Variables**：与现有主题系统一致

**架构模式：**
- Feature-based 结构：新建 `features/pet/` 目录
- 状态独立：新建 `petStore` 与现有 store 并存
- 低耦合：通过事件与主应用通信

## 二、详细开发计划

### Phase 1: 基础架构

#### 1.1 创建目录结构
```
src/features/pet/
├── components/
│   ├── PetCompanion.tsx      # 桌宠主组件
│   ├── PetDocked.tsx         # 停靠态视图
│   ├── PetExpanded.tsx       # 展开态视图
│   ├── MoodBubble.tsx        # 灵感气泡
│   ├── WritingReminder.tsx   # 写作提醒
│   └── QuickActions.tsx      # 快捷操作
├── hooks/
│   ├── usePetMood.ts         # 心情状态管理
│   ├── useWritingStats.ts    # 写作统计
│   └── useInspiration.ts     # 灵感触发
├── store/
│   └── petStore.ts           # 桌宠状态管理
├── types/
│   └── pet.ts                # 类型定义
└── styles/
    └── pet.css               # 样式文件
```

#### 1.2 状态系统设计

**PetMood（心情状态）：**
```typescript
type PetMood = 'focused' | 'excited' | 'thinking' | 'tired' | 'idle';
```

**PetState（组件状态）：**
```typescript
interface PetState {
  mood: PetMood;
  isExpanded: boolean;
  showBubble: boolean;
  bubbleContent: string | null;
  lastActivityTime: number;
  writingSessionStart: number;
  todayWordCount: number;
}
```

### Phase 2: 核心组件

#### 2.1 PetCompanion（主组件）
- 三种形态：停靠态 / 陪写态 / 展开态
- 位置：固定在右下角或侧边栏边缘
- 响应状态变化动画

#### 2.2 心情状态逻辑
| 用户行为 | 心情状态 | 触发条件 |
|---------|---------|---------|
| 持续输入 | 专注 focused | 输入频率 > 3次/10秒 |
| 完成章节 | 兴奋 excited | 保存章节完成 |
| 长时间停顿 | 思考 thinking | 停顿 > 60秒 且 < 5分钟 |
| 长时间无操作 | 困倦 tired | 停顿 > 5分钟 |
| 正常状态 | 空闲 idle | 默认状态 |

#### 2.3 灵感气泡触发
- 卡文检测：停顿 > 90秒
- 气泡内容：短促高价值提示（10-20字）
- 可配置关闭频率

### Phase 3: 交互功能

#### 3.1 快捷操作入口
- 继续写
- 展开 AI 面板
- 总结本章
- 灵感收藏

#### 3.2 展开态面板
- 写作统计仪表盘
- 灵感历史记录
- 快速问答入口

## 五、风险与控制

| 风险 | 缓解措施 |
|-----|---------|
| 桌宠遮挡编辑器 | 提供位置配置和折叠选项 |
| 状态变化过于频繁 | 设置状态切换冷却时间（≥30秒） |
| 灵感气泡打扰用户 | 默认关闭，可手动开启 |
| 性能影响 | 使用 React.memo 和 useMemo 优化 |

## 六、验收标准

1. 桌宠组件可独立运行，不影响现有功能
2. 心情状态能根据写作行为正确切换
3. 灵感气泡可触发并显示
4. 展开态面板显示正确的写作统计
5. 可通过配置关闭桌宠功能
6. 与现有主题系统（暗黑/明亮）兼容
