# 桌宠化AI写作助手（墨灵）开发计划

## 一、架构分析

### 1.1 创意方案与现有架构的契合点

| 创意方案功能 | 现有架构支撑 | 实现方式 |
|-------------|-------------|---------|
| 桌宠停靠组件 | MainLayout + CSS Grid | 新增 PetLayer 覆盖层 |
| 状态系统 | Zustand store | 新建 petStore |
| AI 会话能力 | lib/ai.ts + AIPanel | 复用现有 AI 能力 |
| 章节状态 | useChapterManager | 监听章节变化触发桌宠行为 |
| 写作统计 | StatusBar + wordCount | 复用或扩展 |
| 灵感气泡 | 新建组件 | Popup/Bubble 机制 |
| 文本导出 | useChapterManager + Tauri FS | 新建导出模块 |

### 1.2 技术决策

**技术选型理由：**
- **React 19** + **TypeScript**：与现有项目一致
- **Zustand**：现有项目已使用，便于状态共享
- **Ant Design 6**：现有项目已使用，继续使用其组件
- **CSS Variables**：与现有主题系统一致
- **docx npm 包**：用于 Word 文档生成

**架构模式：**
- Feature-based 结构：新建 `features/pet/` 和 `features/export/` 目录
- 状态独立：新建 `petStore` 与现有 store 并存
- 低耦合：通过事件与主应用通信

src/features/pet/

#### 1.2 状态系统设计

**PetMood（心情状态）：**
```typescript
type PetMood = 'focused' | 'excited' | 'thinking' | 'tired' | 'idle';
```

**PetState（组件状态）：**
```typescript
interface PetState {
  mood: PetMood;
  isExpanded: boolean;
  showBubble: boolean;
  bubbleContent: string | null;
  lastActivityTime: number;
  writingSessionStart: number;
  todayWordCount: number;
}
```

### Phase 2: 核心组件

#### 2.1 PetCompanion（主组件）
- 三种形态：停靠态 / 陪写态 / 展开态
- 位置：固定在右下角或侧边栏边缘
- 响应状态变化动画

#### 2.2 心情状态逻辑
| 用户行为 | 心情状态 | 触发条件 |
|---------|---------|---------|
| 持续输入 | 专注 focused | 输入频率 > 3次/10秒 |
| 完成章节 | 兴奋 excited | 保存章节完成 |
| 长时间停顿 | 思考 thinking | 停顿 > 60秒 且 < 5分钟 |
| 长时间无操作 | 困倦 tired | 停顿 > 5分钟 |
| 正常状态 | 空闲 idle | 默认状态 |

#### 2.3 灵感气泡触发
- 卡文检测：停顿 > 90秒
- 气泡内容：短促高价值提示（10-20字）
- 可配置关闭频率

### Phase 3: 交互功能

#### 3.1 快捷操作入口
- 继续写
- 展开 AI 面板
- 总结本章
- 灵感收藏

#### 3.2 展开态面板
- 写作统计仪表盘
- 灵感历史记录
- 快速问答入口

### Phase 4: 文本保存与导出功能

#### 4.1 修复保存状态显示

**问题描述**：右下角保存指示灯一直显示"未保存"，用户困惑

**解决方案**：
```typescript
// 状态同步机制
interface SaveState {
  status: 'saved' | 'saving' | 'unsaved';
  lastSavedTime: number;
  pendingChanges: boolean;
}
```

**关键修复点**：
1. 确保 `useChapterManager` 的 `saveStatus` 正确同步到 `StatusBar`
2. 添加状态转换的边界条件检查
3. 确保 `creatorai:saveStatus` 事件正确广播

#### 4.2 文件位置可见

**功能描述**：让用户能够方便地找到保存的文件

**实现方式**：
```typescript
// 新增命令：打开章节文件所在目录
async function openChapterDirectory(projectPath: string, chapterId?: string): Promise<void>
```

**UI 入口**：
- 状态栏添加"打开文件夹"图标按钮
- 侧边栏章节列表添加右键菜单"打开所在文件夹"

#### 4.3 多格式导出功能

**目录结构**：
```
src/features/export/
├── components/
│   ├── ExportDialog.tsx       # 导出对话框
│   ├── ExportOptions.tsx      # 导出选项配置
│   └── ExportProgress.tsx     # 导出进度显示
├── hooks/
│   ├── useExport.ts           # 导出逻辑 Hook
│   └── useExportFormats.ts    # 支持的格式列表
├── utils/
│   ├── exportTxt.ts           # TXT 导出工具
│   ├── exportWord.ts          # Word 导出工具
│   └── exportPdf.ts           # PDF 导出工具
├── store/
│   └── exportStore.ts         # 导出状态管理
├── types/
│   └── export.ts              # 导出相关类型
└── index.ts                   # 模块导出
```

**导出格式定义**：
```typescript
type ExportFormat = 'txt' | 'word' | 'pdf';

interface ExportOptions {
  format: ExportFormat;
  scope: 'single' | 'multiple' | 'all';
  chapterIds?: string[];
  includeTitle: boolean;       // 包含章节标题
  lineBreak: 'empty_line' | 'separator';  // 章节分隔方式
  encoding?: 'utf8' | 'gbk';   // 仅 TXT 格式
}

interface ExportResult {
  success: boolean;
  filePath?: string;
  error?: string;
}
```

**技术方案**：

| 格式 | 实现方式 | 技术依赖 |
|------|---------|---------|
| TXT | 前端直接生成字符串，通过 Tauri 保存 | 无额外依赖 |
| Word | 前端使用 `docx` npm 包生成 DOCX | `docx` 包 (~50KB) |
| PDF | Tauri 端调用系统打印 API 或使用 Rust 库 | `printpdf` 或 WebView 打印 |

## 五、风险与控制

| 风险 | 缓解措施 |
|-----|---------|
| 桌宠遮挡编辑器 | 提供位置配置和折叠选项 |
| 状态变化过于频繁 | 设置状态切换冷却时间（≥30秒） |
| 灵感气泡打扰用户 | 默认关闭，可手动开启 |
| 性能影响 | 使用 React.memo 和 useMemo 优化 |
| 保存状态不同步 | 添加状态转换日志，便于调试 |
| 导出文件编码问题 | 统一使用 UTF-8，界面提示编码设置 |

## 六、验收标准

### 桌宠功能验收标准

1. 桌宠组件可独立运行，不影响现有功能
2. 心情状态能根据写作行为正确切换
3. 灵感气泡可触发并显示
4. 展开态面板显示正确的写作统计
5. 可通过配置关闭桌宠功能
6. 与现有主题系统（暗黑/明亮）兼容

### 文本保存与导出验收标准

1. 右下角保存指示灯状态准确反映实际保存状态
2. 用户可通过点击按钮打开章节文件所在目录
3. TXT 导出功能可正常导出单章和全本
4. Word 导出功能生成标准 DOCX 文件
5. PDF 导出功能生成排版整洁的 PDF 文件
6. 批量导出支持选择章节范围
7. 导出过程有进度提示
8. 导出完成后可一键打开目标文件夹
