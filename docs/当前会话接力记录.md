## 最近已完成的重要修复

### 1. Windows 安装版 AI 引擎启动失败

现象：

- 源码环境能用
- 安装 MSI 后调用 AI 时失败
- 报错包括：
  - `AI 引擎启动失败`
  - `ai-engine exited unexpectedly`
  - `EOF`

已经处理：

- 修复 sidecar 查找与打包路径问题
- 修复安装版优先读取错误产物的问题
- 确保安装后能正确启动 `ai-engine`

### 2. 安装后后台频繁弹黑色 shell 窗口

现象：

- 调用 AI 时反复闪黑框

已经处理：

- Windows 子进程改为 `CREATE_NO_WINDOW`
- 安装版运行时不再频繁弹 shell

### 3. 卸载重装后旧聊天界面和旧状态残留

现象：

- 重装后仍带着之前的聊天界面和前端状态

已经处理：

- 后端增加重装清理标记
- 前端消费一次性清理标记
- 清理 `creatorai:*` 相关本地存储
- 避免误删当前版本运行目录

### 4. 编辑器文本输入一段时间后内容消失

现象：

- 用户持续写作约一分钟后，文本会被旧状态覆盖或直接消失

根因：

- 自动保存成功后，父组件未同步最新内容
- 编辑器在特定时机又拿旧的 `initialContent` 回写

已经处理：

- 保存成功后同步：
  - `setChapterContent(content)`
  - `setDraftContent(content)`

### 5. 编辑器快捷键体验差

用户明确要求支持并优化：

- `Ctrl+S`
- `Ctrl+Z`
- `Ctrl+Y`
- `Ctrl+Shift+Z`
- `Ctrl+A`

已经处理：

- 显式绑定常用快捷键
- 修正文案提示
- 新增快捷键交互回归测试

## 用户偏好与工作习惯

这位用户的风格很明确：

- 更关注结果，不喜欢空谈
- 希望开发动作能直接落地
- 很看重“测试要固化成规范”
- 强调独立测试工程，而不是零散脚本
- 希望文档写清楚，方便以后换模型继续接手
- 喜欢有产品感、有陪伴感、有辨识度的设计
- 对 UI 有要求，不接受普通后台工具感

所以后续继续做时应遵守：

- 每做一个重要修复，都补测试
- 每做一个重要修复，都补文档
- 有发布动作时要先验证再打包
- 涉及 API key、安全、安装包的问题要格外保守
