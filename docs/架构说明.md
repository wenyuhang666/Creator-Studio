# Creator Studio 架构说明

## 一、整体架构概览

Creator Studio 是一个基于 Tauri 的桌面应用，采用前后端分离架构：

```
┌─────────────────────────────────────────────────────────────┐
│                     Frontend (React)                         │
│                                                               │
│  ┌─────────┐   ┌─────────┐   ┌─────────┐   ┌─────────┐    │
│  │Layouts  │ → │Features │ → │ Hooks   │ → │ Platform│    │
│  └─────────┘   └─────────┘   └─────────┘   └─────────┘    │
│       ↑            ↑            ↑            ↑               │
│  ┌─────────────────────────────────────────────────┐       │
│  │              Components (UI Components)          │       │
│  └─────────────────────────────────────────────────┘       │
└──────────────────────────┬──────────────────────────────────┘
                           │ Tauri IPC (invoke/events)
┌──────────────────────────┼──────────────────────────────────┐
│                     Backend (Rust)                           │
│                                                               │
│  ┌─────────────────────────────────────────────────┐       │
│  │                  Command Handlers                 │       │
│  │     (project, chapter, session, config, rag...)   │       │
│  └─────────────────────────────────────────────────┘       │
│  ┌─────────────────────────────────────────────────┐       │
│  │                 Business Logic                    │       │
│  │     (ai_bridge, file_ops, keyring_store...)      │       │
│  └─────────────────────────────────────────────────┘       │
│  ┌─────────────────────────────────────────────────┐       │
│  │               AI Sidecar (Node.js)               │       │
│  └─────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

## 二、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端框架** | React 19 | UI 渲染 |
| **类型系统** | TypeScript | 类型安全 |
| **UI 组件库** | Ant Design 6 | 企业级组件 |
| **状态管理** | Zustand | 轻量级状态管理 |
| **桌面框架** | Tauri 2 | Rust 后端 + WebView |
| **后端语言** | Rust | 高性能、安全 |
| **AI 引擎** | Node.js | 本地 sidecar 进程 |

### 3.2 分层职责

#### 应用层 (app/)

负责应用整体状态和初始化流程：

| 文件 | 职责 |
|------|------|
| `AppProviders.tsx` | 组合 Ant Design ConfigProvider、ThemeProvider |
| `bootstrap.ts` | 启动时加载最近项目、检查清理标志 |
| `store.ts` | 定义 `useAppStore`，管理当前项目、最近项目 |
| `types.ts` | 应用级别的 TypeScript 类型 |

**AppProviders.tsx 示例**：
```tsx
export function AppProviders({ children }: { children: ReactNode }) {
  const { theme, updateTheme } = useTheme();
  
  return (
    <ConfigProvider theme={themeConfig}>
      <ThemeProvider value={{ theme, updateTheme }}>
        {children}
      </ThemeProvider>
    </ConfigProvider>
  );
}
```

**useAppStore 示例**：
```typescript
interface AppState {
  currentProject: ProjectConfig | null;
  recentProjects: RecentProject[];
  loadRecentProjects: () => Promise<void>;
  openProject: (path: string) => Promise<void>;
  // ...
}
```

#### 平台层 (platform/tauri/)

隔离业务代码与 Tauri 依赖，便于后续迁移：

| 文件 | 职责 |
|------|------|
| `client.ts` | 封装所有 `invoke` 调用，提供类型安全的命令调用 |
| `events.ts` | 定义事件常量、类型，封装事件发布/订阅 |
| `storage.ts` | 封装 localStorage/sessionStorage 操作 |

**client.ts 核心结构**：
```typescript
// 类型定义
export interface Provider {
  id: string;
  name: string;
  base_url: string;
  models: string[];
  // ...
}

// 命令封装
export async function openProject(path: string): Promise<ProjectConfig> {
  return tauriInvoke("open_project", { path });
}

export async function listChapters(projectPath: string): Promise<ChapterMeta[]> {
  return tauriInvoke("list_chapters", { projectPath });

```

**events.ts 核心结构**：
```typescript
// 事件类型
export const EVENTS = {
  CHAPTER_SELECTED: "creatorai:chapterSelected",
  CHAPTERS_CHANGED: "creatorai:chaptersChanged",
  SAVE_STATUS: "creatorai:saveStatus",
  GLOBAL_CONFIG_CHANGED: "creatorai:globalConfigChanged",
} as const;

// 发布事件
export function emitChapterSelected(chapterId: string): void {
  window.dispatchEvent(new CustomEvent(EVENTS.CHAPTER_SELECTED, { 
    detail: { chapterId } 
  }));
}
```

### 6.3 事件驱动

使用 CustomEvent 实现组件间通信：

```typescript
// 发布事件
export function emitChapterSelected(chapterId: string): void {
  window.dispatchEvent(
    new CustomEvent("creatorai:chapterSelected", { 
      detail: { chapterId } 
    })
  );
}

// 订阅事件
useEffect(() => {
  const handler = (e: CustomEvent) => {
    loadChapter(e.detail.chapterId);
  };
  window.addEventListener("creatorai:chapterSelected", handler);
  return () => window.removeEventListener("creatorai:chapterSelected", handler);
}, [loadChapter]);
```

**核心事件列表**：

| 事件名 | 触发时机 | 订阅者 |
|--------|----------|--------|
| `creatorai:chapterSelected` | 用户选择章节 | Editor, AIPanel |
| `creatorai:chaptersChanged` | 章节列表变更 | Sidebar |
| `creatorai:chapterAppended` | AI 追加内容 | Editor |
| `creatorai:saveStatus` | 保存状态变更 | StatusBar |
| `creatorai:globalConfigChanged` | 配置变更 | AIPanel |
| `creatorai:openSettings` | 打开设置 | SettingsPanel |

## 八、架构原则

### 8.1 前端原则

1. **平台隔离**：业务代码不直接依赖 Tauri API，通过 `platform/` 层封装
2. **状态集中**：使用 Zustand 管理状态，避免散落的 useState
3. **事件类型化**：使用常量定义事件名，避免字符串硬编码
4. **组件拆分**：超过 200 行的组件应考虑拆分

### 8.2 后端原则

1. **命令单一职责**：每个命令只做一件事
2. **错误传播**：使用 `?` 操作符传播错误
3. **状态共享**：使用 Arc<Mutex<>> 共享状态
4. **安全优先**：所有路径操作需校验

## 九、待改进项

### 高优先级

1. **Rust 层分层**：`lib.rs` (~1000行) 需要拆分
   - commands/ - 命令处理
   - application/ - 业务逻辑
   - domain/ - 领域模型
   - infrastructure/ - 基础设施

2. **AI Bridge 模块化**：`ai_bridge.rs` (~1200行) 需要拆分
   - protocol.rs - 协议定义
   - runtime.rs - 运行时管理
   - tool_executor.rs - 工具执行

### 中优先级

3. **前端单元测试**：为 store、hooks 补充 Vitest 测试
4. **错误处理统一**：Rust 端用 anyhow 或 thiserror 替代 String
5. **类型导出**：统一管理 TypeScript 类型导出

### 低优先级

6. **组件文档**：为关键组件添加 Storybook
7. **性能监控**：添加首屏加载、API 响应时间监控
