# 桌宠化 AI 写作助手创意方案

## 背景

当前的 Creator Studio 已经具备：

- 章节编辑
- AI 对话与续写
- 默认 Provider 配置
- 回归测试与打包链路

下一阶段可以往“更有陪伴感、更有记忆点、更符合长期写作者习惯”的方向升级，不只是一个工具，而是一个会陪用户写作的桌面伙伴。

参考项目：
- MuMuAINovel
- 参考仓库：`https://github.com/xiamuceer-j/MuMuAINovel`

基于该项目公开 README 可提炼出的方向有：
- 多模型与多创作模式
- 世界观、角色、章节等结构化资产
- 可视化内容管理
- 更重产品感和展示感的 UI

这里进一步往“桌宠 + 写作助手”方向扩展。

## 产品定位

把 Creator Studio 从“AI 写作软件”升级成“会陪你写作的桌面创作精灵”。

核心不是单纯卖萌，而是：

- 降低创作孤独感
- 把常用能力做成低打扰、高频率的桌面交互
- 用更自然的方式推动写作节奏
- 让用户对软件形成情感记忆点

## 名字建议

推荐名字：`墨灵`

原因：

- “墨”直接对应写作、文字、创作
- “灵”有灵感、机灵、精灵感，也适合桌宠人格
- 两个字短，好记，适合按钮、标题、品牌角标、桌宠称呼
- 不幼稚，既能做可爱风，也能做高级感视觉

可作为产品内角色名：

- 墨灵
- 墨灵助手
- 小墨灵

备选：

- `字灵`
- `灵墨`
- `叙灵`
- `灵页`

如果要一个更偏品牌的组合名，推荐：

- `Creator Studio · 墨灵`

## 核心设想

在应用中加入一个“桌宠型 AI 助手”，它不是一直强打扰的悬浮物，而是一个可切换状态的智能陪伴层。

三种存在形态：

1. 停靠态
- 停在编辑器边缘、右下角、侧栏顶部
- 占空间很小，只显示表情、状态、未读提示

2. 陪写态
- 在用户写作时偶尔给出鼓励、提醒、剧情线索提示
- 可主动弹出“要不要继续这一段”“这一章节奏有点平，要不要我帮你推一下”

3. 展开态
- 展开成一个完整的 AI 助手面板
- 可做对话、续写、角色问答、总结、复盘、任务拆解

## 适合做的有意思功能

### 1. 陪写桌宠

- 桌宠有心情状态：专注、兴奋、思考、困倦、提醒
- 根据用户行为变换状态：
  - 长时间输入时显示“专注”
  - 停顿太久时显示“思考”
  - 连续完成章节时显示“兴奋”
- 状态变化不要太频繁，避免吵

### 2. 灵感掉落

- 当用户卡文超过一段时间，桌宠掉出一个“灵感气泡”
- 气泡内容不是长篇回复，而是短促高价值提示：
  - “要不要让配角先说破这个秘密？”
  - “这段可以加一个动作细节，会更有画面感。”
  - “这一章现在缺一个钩子。”

### 3. 写作节奏提醒

- 检测用户连续写作时长
- 提供轻量提醒：
  - 25 分钟专注完成
  - 该保存了
  - 这一章已经超过目标字数
  - 今天已经完成 1800 字

### 4. 情绪化反馈

- 用户完成章节、保存成功、解决 bug、导出文本时，桌宠给仪式感反馈
- 比如：
  - “这一章站住了。”
  - “这一段情绪很稳。”
  - “你今天的推进比昨天快。”

这个功能很适合提升留存。

### 5. 世界观问答

- 用户可以像问一个懂自己作品的角色一样去问桌宠
- 例如：
  - “主角第三章知道真相了吗？”
  - “林策和白檀现在关系变化到哪一步了？”
  - “这个设定和前文冲突吗？”

这类能力和现有知识库、章节读取、总结系统天然契合。

### 6. 角色扮演模式

- 桌宠可以切换身份
- 例如：
  - 编辑模式
  - 毒舌读者模式
  - 温柔陪写模式
  - 世界观顾问模式
  - 角色扮演模式

用户对“一个会切人格的桌宠”会更有新鲜感。

### 7. 写作任务看板

- 桌宠可以给出每日任务
- 例如：
  - 完成本章开头
  - 补完反派动机
  - 修掉本章 3 个重复表达
  - 回收一个伏笔

做成小卡片，比纯文本 AI 回复更像产品。

### 8. 剧情温度计

- 用可视化方式告诉用户当前章节的节奏和情绪
- 例如：
  - 紧张度
  - 信息密度
  - 人物冲突度
  - 情绪波峰

这是非常适合“漂亮 UI”的点。

### 9. 收藏式灵感仓库

- 用户看到桌宠给出的好点子，可以一键收藏
- 收藏内容自动分类：
  - 情节
  - 台词
  - 角色
  - 设定
  - 伏笔

### 10. 桌宠成长系统

- 不要做重游戏化数值
- 只做轻成长：
  - 陪写天数
  - 完成章节数
  - 累计字数
  - 共同完成作品数

成长的结果是解锁：
  - 新表情
  - 新动作
  - 新主题皮肤
  - 新语气包

## 更切合用户习惯的交互原则

### 低打扰

- 默认小体积停靠
- 不要频繁自动弹窗
- 不抢输入焦点
- 所有主动打断都要可关闭

### 强即时性

- 常见操作必须一键直达
- 例如：
  - 继续写
  - 展开剧情建议
  - 总结本章
  - 查看人物卡
  - 保存草稿

### 短反馈优先

- 桌宠先给一句短反馈
- 用户点开再看长内容
- 这样比默认铺一大片 AI 文本更符合写作者节奏

### 陪伴优先于表演

- 不要把桌宠做成无意义动画挂件
- 每次出现最好都对创作有帮助

## UI 方向

目标不是普通后台管理界面，而是“创作工作台 + 轻幻想陪伴层”。

### 视觉气质

- 温暖、精致、安静
- 不要过度二次元，也不要纯商务后台风
- 更像高质量写作器物

### 推荐风格

- 主色：
  - 墨黑
  - 米白
  - 琥珀金
  - 雾青
  - 纸张暖灰

- 材质：
  - 半透明玻璃卡片
  - 纸张纹理背景
  - 柔和投影
  - 微弱发光边缘

### 重点 UI 模块

1. 桌宠停靠卡片
- 圆角大一点
- 有表情、状态字、快捷按钮
- 可折叠

2. 灵感气泡
- 类似即时消息气泡
- 可以收藏、忽略、展开

3. 创作仪表板
- 今日字数
- 本章进度
- 最近灵感
- 角色冲突提示

4. 情绪与节奏图
- 用曲线、热力条、章节节点图
- 增强“创作被看见”的感觉

5. 主题皮肤系统
- 纸墨风
- 夜航风
- 古典书房风
- 轻科幻写作舱风

## 可以补充的漂亮 UI 功能

- 专注模式
  - 收起多余边栏
  - 桌宠缩成静音陪伴态

- 夜间陪写灯
  - 夜间模式下桌宠周围有柔和光圈

- 章节封面卡
  - 每章可自动生成一句标题副标和视觉封面色

- 角色关系图谱
  - 节点可拖拽
  - 点角色可直接问桌宠“这个人最近的动机是什么”

- 伏笔时间线
  - 让剧情伏笔可视化
  - 这点很适合和桌宠联动做提醒

- 灵感雨
  - 特殊事件触发的小动画
  - 比如完成章节时掉出几条关键词

- 成就陈列柜
  - 完结章节
  - 连续创作天数
  - 写作里程碑

## 第一阶段建议先做什么

先做轻量版本，不要一上来做复杂养成。

### Phase 1

- 增加桌宠停靠组件
- 增加 5 种状态
- 增加灵感气泡
- 增加快捷操作入口
- 增加陪写提醒

### Phase 2

- 增加角色人格切换
- 增加剧情温度计
- 增加任务卡片
- 增加灵感收藏夹

### Phase 3

- 增加成长系统
- 增加主题皮肤
- 增加更强的图谱型可视化

## 技术落地方向

结合当前项目结构，比较合理的做法是：

- 前端增加一个全局 `PetAssistantLayer`
- 停靠在 `MainLayout` 之上，但不盖住主编辑器核心区域
- 与现有 AI 会话、章节、总结、知识库状态打通
- 所有主动提醒都走可配置策略

建议新增状态源：

- 当前章节字数
- 最近保存时间
- 最近输入时间
- 当前写作会话时长
- 当前 AI 会话模式
- 当前章节是否存在总结

## 文案风格建议

桌宠说话要像“懂写作的搭档”，不要像客服。

推荐语气：

- 简短
- 有温度
- 有判断
- 不浮夸

示例：

- “这一段气口很好。”
- “你卡的不是文字，是转折。”
- “这一章差一个更狠的决定。”
- “我觉得这里可以让配角先出手。”

不推荐：

- 过度卖萌
- 连续感叹号
- 太长的说教
- 像客服提示框

## 结论

这个方向值得做，而且很适合 Creator Studio。

因为你现在这个产品不是缺一个普通功能，而是缺一个真正让用户记住的“人格化入口”。  
“墨灵”如果做对了，会同时提升：

- 产品辨识度
- 用户停留时长
- 写作陪伴感
- UI 质感
- AI 功能使用率

一句话版本：

把 AI 从侧边栏工具，变成一个安静、聪明、漂亮、真的懂创作节奏的桌面写作精灵。
